Putting Your Content in a Grid Container

Before we can define a grid in CSS, we need to nest all of the content inside an id that we will attach to a div. Although we could apply these settings directly to the body, we'd limit our flexibility for changing layout down the line - all of our content would be forced to exist inside our CSS grid, whether we wanted it to or not. To avoid that situation, by default we will contain all of our grid content in a div with an id name of grid.

The Display Property

In CSS, we default to displaying content in boxes. To change that, we'll invoke the display: grid; property to tell the browser that instead of stacking HTML boxes in flow sequence, we're going to place each Box Model element on a defined screen grid. This gives us tremendous control over Web page layout. That used to be more relevant a few years ago when it was still common to see complex layouts, but even in the mobile-first era of Web design we can make good use of CSS Grid.

Alternatives to CSS Grid

  • HTML Flow
  • Floats
  • Flexbox

HTML flow is the original layout tool, before we had CSS. With HTML Flow all you can do is structure your HTML in the order you want content to appear on the finished Web page. It is limited to a single column unless you use <table> tags to mimic a grid. Overall, HTML flow is to be used only as a backup for those cases in which your CSS completely fails to load; it should never be your primary tool for layout.

Floats were intended to replace tables, but floats struggled to support anything more than simple two-column layouts. Floats are finicky and were a bad fit for the kinds of complex Web layouts that became popular about 10 years ago.

Flexbox replaced floats (for the most part - some designers still include floats as part of their Flexbox or CSS Grid layout as a sort of hybrid design) but Flexbox really shines when you are tiling a lot of items in a group. Flexbox struggles with full-page layout control. The best designers today begin with CSS Grid and then insert Flexbox to handle a group of items within the layout.
